# UML : LES DIAGRAMMES
## Diagramme de cas d'utilisation
Il cartographie les fonctionnalités du système du point de vue des utilisateurs. Il montre en un coup d'œil quelles actions chaque acteur (Membre, Bibliothécaire) a le droit d'effectuer.
## Diagramme de classe
Il définit l'architecture et la structure de la base de données de la bibliothèque. Il liste les entités (Membres, Livres, Catalogues), leurs données et la façon dont elles sont reliées.
## Diagramme de sequence
Il montre la chronologie des échanges pour une action précise, ici l'emprunt d'un livre. Il détaille étape par étape comment l'interface, le membre et le système communiquent dans le temps.
## Diagramme d'etat transition
Il retrace le cycle de vie d'un exemplaire de livre au sein de la bibliothèque. Il montre comment un ouvrage passe d'un état à un autre (disponible, réservé, emprunté ou perdu) selon les situations.
